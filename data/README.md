# Data

**None of the raw data is committed** — the Chandrayaan-2 DFSAR scenes and the LOLA
DEM together exceed 12 GB, and individual files run to 0.6–1.3 GB (GitHub hard-rejects
files over 100 MB). This file explains how to obtain each input and where to place it so
the pipeline scripts find it. All paths are relative to the repository root and are
configured in [`config/config.yaml`](../config/config.yaml).

```
data/
├── raw/          # downloaded source data (git-ignored)
│   ├── lola/     # LOLA DEM + slope rasters
│   └── dfsar/    # Chandrayaan-2 DFSAR scenes, derived mosaics, geometry, ice catalogue
├── interim/      # cropped AOIs / derived rasters written by scripts 00–03 (git-ignored)
└── processed/    # small router-input CSVs written by script 07 (committed as examples)
```

## 1. LOLA topography (NASA PGDA)

South-polar LOLA DEM and surface-slope rasters (Barker et al. 2021), 5 m/pixel,
south polar stereographic on the 1737.4 km lunar sphere.

- **Source:** NASA Planetary Geodynamics Data Archive (PGDA) — LOLA south-pole products:
  <https://pgda.gsfc.nasa.gov/products/78>
- **Files and destinations** (see `config.yaml: paths`):
  - `data/raw/lola/ldem_87s_5mpp.tif`  — surface height Z (metres)
  - `data/raw/lola/ldsm_87s_5mpp.tif`  — surface slope (degrees)

These are the whole south pole; the pipeline window-reads only a ±15 km AOI around the
target crater, so you never load the full raster into memory.

## 2. Chandrayaan-2 DFSAR (ISRO PRADAN / ISSDC)

Dual-Frequency SAR L-band products, distributed by ISRO through the PRADAN portal of the
ISRO Science Data Archive (ISSDC):

- **Source:** <https://pradan.issdc.gov.in/ch2/> (DFSAR — free registration required)

Two DFSAR products are used:

**(a) L3C derived south-pole-east mosaic** — CPR + four-component decomposition
(`vol`/`odd`/`evn`/`hlx`), 25 m/pixel (script `06_dfsar_detect.py`). Place under
`data/raw/dfsar/data/derived/20250630/` (filenames in `config.yaml: dfsar`).

**(b) Calibrated full-pol SLI scene `20200321t082617351`** — the four complex channels
(HH/HV/VH/VV, Band1=I, Band2=Q) plus the `g_sli` geometry CSV and `sri_ma` valid-data
mask (script `08_fp_detect.py`). Place under
`data/raw/dfsar/data/calibrated/20200321/` and
`data/raw/dfsar/geometry/calibrated/20200321/` (filenames in `config.yaml: fp`).

**(c) Polar water-ice catalogue** `ICY_CRATERS_SP.tif` — used only for cross-validation.
Place at `data/raw/dfsar/DFSAR_Water_Ice/Polar_Water_Ice_Products/ICY_CRATERS_SP.tif`.

## Notes

- The exact filenames the scripts expect are listed in `config/config.yaml`; if your
  download names differ, update the config rather than renaming files.
- `data/interim/` is regenerated by the pipeline and is safe to delete.
- The small CSVs in `data/processed/` (`candidates.csv`, `candidates_router.csv`,
  `depot.csv`) are committed as ready-made examples of the detection → traverse hand-off.
